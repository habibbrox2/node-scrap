API Reference
Base URL: http://localhost:9999
GET /api/status
Current scraper status + cache stats + active cron jobs
POST /api/scrape
Manually trigger a scrape (runs in background)
GET /api/articles?page=1&limit=20&category=&search=
Paginated cached articles with optional filters
GET /api/mobiles?page=1&limit=20&brand=&status=&search=
Paginated cached mobiles with optional filters
GET /api/articles/search?url=https://...
Find a specific article by its URL
GET /api/mobiles/search?url=https://...
Find a specific mobile by its URL
GET /api/stats
Detailed statistics and run history
GET /api/cache/export
Download full cache as JSON file
DELETE /api/cache
Clear all cached articles
POST /api/cron body: { action, schedule, name }
Manage cron jobs — action: start | stop | list
GET /api/scrape/log
Get live scrape log entries
Settings + Delivery
GET /api/settings
Read API settings (push endpoint + headers)
POST /api/settings
Update API settings (pushEndpointUrl, pushEndpointHeadersJson)
After each scrape run (manual/cron), items are saved to JSON cache and also stored in a local SQLite DB. If a push endpoint is configured, newly added items are POSTed to it as: { source, trigger, pushedAt, count, articles: [...] }.

Not saved

curl http://localhost:9999/api/status

curl -X POST http://localhost:9999/api/settings -H "Content-Type: application/json" -d '{\"pushEndpointUrl\":\"https://broxlab.online/api/autocontent\",\"pushEndpointHeadersJson\":\"{\\\\\\\"Authorization\\\\\\\":\\\\\\\"Bearer ...\\\\\\\"}\"}'
curl "http://localhost:9999/api/articles?page=1&limit=10&search=ক্রিকেট"
curl "http://localhost:9999/api/mobiles?page=1&limit=10&brand=Honor"
curl -X POST http://localhost:9999/api/cron \
  -H "Content-Type: application/json" \
  -d '{"action":"start","schedule":"0 */3 * * *","name":"every-3h"}'
curl http://localhost:9999/api/cache/export -o articles.json
